# v1.0.0 — Long live teletext!

First stable release. Three npm packages and one Claude Code plugin, all locked at 1.0.0. The teletext that powered Sunday-morning weather checks since the eighties is now something Claude can read for you on demand.

## Highlights

- **Universal install.** Works in Claude Code, Claude Desktop, Cursor, VS Code Copilot Chat, Continue, Goose, Windsurf, Cline, and Codex CLI — anything that speaks MCP. Plus a plain CLI for shells, scripts, and CI jobs.
- **Two broadcasters shipped.** [ČT Teletext](https://www.ceskatelevize.cz/teletext/) (Czechia) and [SVT Text](https://www.svt.se/text-tv) (Sweden). PRs for more are welcome and take roughly 30 minutes.
- **Provider-modular core.** Adding a new broadcaster is one file in `packages/core/src/providers/` plus a one-line registry edit. Everything else (CLI, MCP, slash commands) picks it up automatically.
- **Platform-aware cache.** Lands in the right place on every OS: `~/Library/Caches/teletext/` on macOS, `%LOCALAPPDATA%\teletext\Cache\` on Windows, `$XDG_CACHE_HOME/teletext/` on Linux.
- **Zero runtime dependencies.** Everything compiles to plain Node ESM. The MCP server, CLI, and core library together pull in nothing at install time.
- **Offline-friendly Claude Code plugin.** The plugin ships a pre-bundled MCP server in `plugin/dist/` — no `npx` cold-start, no network round-trip when you launch Claude Code.

## Install

**Claude Code** — get slash commands *and* the MCP tools:

```text
/plugin marketplace add honzanemecek/teletext
/plugin install teletext@honem
```

**Any other MCP-capable client** — add this to the host's config:

```json
{
  "mcpServers": {
    "teletext": {
      "command": "npx",
      "args": ["-y", "@honem/teletext-mcp"]
    }
  }
}
```

**Shell, script, CI** — no install:

```sh
npx -y @honem/teletext-cli list
npx -y @honem/teletext-cli --provider=ct 100
```
